## Pregunta 2

### Compromisos del grupo - Kernel Panic 404

Como equipo, acordamos los siguientes compromisos para trabajar de manera colaborativa y organizada durante el semestre:

**Organización con metodologías ágiles:**
- Usaremos un tablero Kanban en GitHub Projects para organizar las tareas en columnas: Por hacer, En progreso y Terminado.
- Cada tarea debe tener un responsable asignado antes de comenzar.

**Reuniones y comunicación:**
- Nos reuniremos mínimo una vez por semana de forma virtual para sincronizar avances.
- Usaremos un grupo de WhatsApp para comunicación diaria y avisos urgentes.

**Uso de Git y GitHub:**
- Cada integrante trabajará en su propia rama y nunca hará cambios directos a `main`.
- Todo cambio a `main` debe hacerse mediante Pull Request y ser revisado por al menos un compañero antes del merge.
- Los mensajes de commit deben ser descriptivos y seguir el formato: `feat:`, `fix:`, `docs:`, etc.

**Responsabilidad:**
- Si algún integrante no puede cumplir una tarea, debe avisar al grupo con al menos 24 horas de anticipación.
- Las decisiones importantes se tomarán por consenso del grupo.
